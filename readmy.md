Документация
------------

TrenagerKoordPlosk - приложение для отработки навыков определения координат 
точек на плоскости для учеников 5-6 классов (тема есть в школьной программе 
по информатике в 5 классе и по математике в 6 классе).


*Настройка программы*

Необходимые для работы приложения модули перечислены в файле requirements.txt
Установка модулей из файла requirements.txt по команде в терминале:
$ pip install -r requirements.txt


*Установка программы*

Не требуется.


*Запуск программы*

Запустить приложение из файла TrenagerKoordPlosk/main.py


**Инструкция**

Приложение предполагает ввод данных пользователя - только фамилию и имя в 
текстовом формате (без цифр и спецсимволов) для внесения в базу данных 
(далее БД) koord_pl.db. В случае использования запрещенных символов во 
всплывающем окне будет сообщено об ошибке.

БД позволяет отслеживать некоторую статистику по пользователю: количество 
запусков приложения и среднюю оценку за выполненные упражнения.

При первичном вводе данных будет выдано сообщение, что запись успешно внесена 
в БД. При повторном входе - сообщение "Рады видеть Вас снова!". 

Приложение предлагает пользователю выбрать рисунок для работы.
Доступен просмотр всех рисунков - выпадающее меню "Выберите рисунок" 
(представлены отличные по уровню сложности рисунки - разное количество точек).

Выбрав рисунок, нужно вводить координаты точек в специальные окна X (координата 
по оси Ох) и Y (координата по оси Оy). 

Начинать обход можно с любой точки рисунка. 

Правильно введенные координаты фиксируются и отображаются списком на экране 
"Введенные координаты", а также подтверждаются сообщением "ОК {x};{y}" 
(вместо х и у отображаются координаты точки). 

При повторном вводе координат ошибка не засчитывается, и на экране Результат 
выводится сообщение "Повтор координат. Введите следующую точку".

Ошибка во введенных координатах отображается на экране Результат сообщением
"Неверно. Попробуйте еще раз". Она фиксируется в программе и влияет на выставление
оценки за работу.

После окончания работы над рисунком на экране Результат появится надпись 
"Все точки перечислены. Нажмите кнопку "Закончить работу". 

Если пользователь не ввел свои данные в начале работы, то его попросят об этом 
во всплывающем окне. 

После завершения работы по кнопке "Закончить работу" во всплывающем окне будет 
сообщено об оценке и о том, что данные о работе успешно внесены в БД.

Если пользователь решил завершить работу раньше, чем перечислил все точки
рисунка, программа выведет сообщение "Вы не закончили упражнение. Чтобы выйти из 
программы, нажмите 'Ok'.Чтобы продолжить, нажмите 'Cancel'".

Если пользователь даже не начал работу над рисунком и решил выйти, то программа
выведет сообщение "Работа не началась. Чтобы выйти из программы, нажмите 'Ok'.
Чтобы продолжить, нажмите 'Cancel'"

В Меню (предназначено для учителя) доступны две вкладки: Файл и О программе.
Во вкладке Файл есть пункты: Добавить файл, Удалить файл и Получить данные.

Добавить файл - вызывает диалговое окно для добавления рисунка из каталога images 
в БД или восстанавливает ранее удаленный рисунок из "корзины" и делает доступным 
выбор этого рисунка в программе сразу после добавления.

Удалить файл - рисунок становится недоступным для выбора в программе 
(удаление в "корзину").

Получить данные - выводит в диалоговом окне ТОП-10 учеников из БД, отсортированных 
по убыванию оценки.

Во вкладке О программе можно найти краткую информацию о приложении и авторе.


*Сообщения об ошибках.*

Приложение находится в стадии активного тестирования пользователями.

Для просмотра корректности результатов работы с БД - файл БД доступен в каталоге 
SQLiteStudio\koord_pl.db и может быть отображен с помощью приложения SQLiteStudio.

Для подготовки собственного рисунка к загрузке в программу TrenagerKoordPlosk 
нужно выполнить два обязательных условия:
1) Файл с рисунком: рисунок должен быть выполнен на координатной плоскости с 
четко отрисованными осями координат и числовой разметкой по осям Ох и Оу. 
Размер рисунка должен быть примерно 505 х 505 пикселей. 
Название рисунка на русском языке, отражает суть изображения. 
Формат рисунка для загрузки - .bmp
2) Файл с координатами точек для проверки правильности работы пользователя:
Название файла совпадает с названием файла с рисунком.
Координата каждой точки записывается в виде х;y и с новой строки.
Например, 
5;7
2;-3
-15;1
Формат файла с координатами - .txt


Использованные технологии
-------------------------
Главное окно приложения выполнено с использованием Qt Designer 
(design_form.ui)

Запросы к БД SQLiteStudio на добавление, удаление, изменение, 
отбор данных, отбор данных с сортировкой для нужд программы

Работа с диалоговыми окнами при выборе пунктов в Меню программы:
выбор файлов, выпадающий динамичный список для удаления рисунка 
и вывод запроса на получение данных в отдельном окне. Обработка 
событий от этих виджетов.

Использование обработчиков разных ошибок с привлечением как
всплывающих окон, так и вывода информации в окне приложения.

Отображение файла с графической информацией в окне приложения и 
масштабирование его под свободную область.

Выгрузка информации из текстового файла с координатами с дальнейшим 
преобразованием и обработкой для контроля работы пользователя.

Чтение данных, вводимых пользователем в приложении, и их обработка.

Работа с разными виджетами - комбобоксом (выбор рисунков), лейблами
(Результат и Введенные координаты), текстовыми полями (ввод данных пользователя
и ввод координат), кнопками (ОК, Проверить, Закончить работу)

Работа с модулем OS для удобства ввода путей к файлам

Исходный код хранится на github. Сделаны коммиты по каждой доработке.
